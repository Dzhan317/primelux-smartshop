#  Ideas y decisiones del proyecto

Aquí se recogen las decisiones tomadas durante el desarrollo y el motivo de cada una.

---

##  Arquitectura

Se utiliza MVC para organizar el código.

Además, se han separado algunas partes en servicios para evitar concentrar toda la lógica en los controladores.

---

##  Seguridad

Se implementa 2FA por correo y almacenamiento seguro de contraseñas.

---

##  Recomendaciones

Se ha optado por un sistema sencillo basado en:

- Categorías  
- Actividad del usuario  

---

##  Limitaciones

Por tiempo, no se han implementado sistemas más complejos.

Se ha priorizado una base funcional.

---

##  Futuro

- Mejoras en recomendaciones  
- Pagos reales  
- Panel de administración
